NarPulse iOS Build Brief

NarPulse — iOS companion app

Native SwiftUI app for residents of the Nərimanov district of Baku. Reads from the same Supabase backend as the web app. Built for the Openwave hackathon — judges will see this side-by-side with the web demo, so it must feel native and polished, not like a wrapper.

Read this first

  • The web app is the primary demo. It has the RİH admin dashboard. Do not duplicate the admin surface on iOS — this app is resident-only.
  • Backend is already built. You consume it. Do not create new tables or modify the schema without confirming first.
  • Default language is Azerbaijani. English is secondary. All strings go through Localizable.strings in az and en bundles.
  • Target: iOS 17+, SwiftUI only, no UIKit unless wrapping MKMapView.
  • Demo device: judge will hold an iPhone. Test on a real phone, not just the simulator.

Storage

Bucket safety-photos is public-read, auth-write. Upload paths follow {user_id}/{uuid}.jpg. Compress to max 1600px wide, JPEG quality 0.7 before upload.

Realtime

Subscribe to postgres_changes on the outages table. On INSERT with status = 'active' within the user's saved district (Nərimanov), surface a banner + (if Phase 5 ships) a local notification.

Screens

1. Home (tabbed root)

Top: greeting + district name + live count of active outages in Nərimanov.
Below: three large tiles — Kəsintilər, Növbələr, Təhlükəsizlik — each with icon, count badge, last-updated timestamp.

2. Outages (Kəsintilər)

Map at top half, list below (or full-map mode toggleable). Pulsing red annotation for active, amber for planned. Tap → detail sheet: area, utility, start time, ETA, description, "Hesabat səhvdir" report button.

3. Wait times (Növbələr)

List of service locations sorted by distance. Each row: name, kind icon, current estimated wait (median of last hour), "Hesabat ver" button. Detail screen: history chart + report sheet with a slider.

4. Safety (Təhlükəsizlik)

Map of pins + FAB to drop a new pin. Tap the FAB → camera/photo picker → category → description → confirm location → submit. Detail: photo, description, upvote button.

5. Account

Sign in with magic link (email), language toggle (AZ/EN), district selector (locked to Nərimanov for v1, prepped for expansion), notifications toggle, "Haqqında" link.

6. Onboarding (first launch)

Three swipeable cards explaining the three features, last card has location permission prompt + "Başla" button. Skip allowed.

Model examples

struct Outage: Identifiable, Codable, Hashable {
    let id: UUID
    let utility: Utility
    let status: OutageStatus
    let areaName: String
    let centerLat: Double
    let centerLng: Double
    let radiusM: Int
    let startedAt: Date
    let estimatedEnd: Date?
    let description: String?

    enum Utility: String, Codable { case water, electricity, gas }
    enum OutageStatus: String, Codable { case planned, active, resolved }
}

extension Outage {
    var coordinate: CLLocationCoordinate2D {
        .init(latitude: centerLat, longitude: centerLng)
    }
    var iconName: String {
        switch utility {
        case .water: "drop.fill"
        case .electricity: "bolt.fill"
        case .gas: "flame.fill"
        }
    }
}

Build phases

Phase 1 — Project setup45 min
  • New Xcode project, iOS 17, SwiftUI app lifecycle, app name NarPulse, bundle id io.narpulse.ios.
  • Add Swift Package: https://github.com/supabase/supabase-swift.
  • Create Theme.swift, force dark mode in NarPulseApp.
  • Config.xcconfig with Supabase URL + anon key; surface via Info.plist.
  • Stub four tabs in a TabView.
Phase 2 — Auth + models30 min
  • Define all four models with Codable + snake_case decoding.
  • Magic-link sign-in screen. Universal Link callback returns to the app and completes auth.
  • Account tab shows email when signed in.
Phase 3 — Outages (the hero)2 h
  • OutageService.fetchActive() + subscribeRealtime().
  • SwiftUI Map with custom annotations. Pulsing red animation via TimelineView + scaleEffect + opacity.
  • Bottom sheet (.sheet with detents .medium, .large) listing outages.
  • Tap annotation → detail sheet.
  • Filter chips at the top: All / Water / Electricity / Gas.
Phase 4 — Wait times1.5 h
  • Fetch service locations + last-hour check-ins; compute median per location.
  • Sort by user distance (request "When in use" location permission with a real NSLocationWhenInUseUsageDescription string in AZ).
  • Report-wait sheet with slider (0–120 min, 5 min steps).
Phase 5 — Safety pins2 h
  • Map of existing pins, FAB → flow: photo (PhotosPicker + camera) → category → description → confirm location (draggable annotation) → submit.
  • Upload photo to safety-photos bucket, store returned URL in safety_pins.photo_url.
  • Detail view with upvote (insert/delete in pin_votes, refresh count).
Phase 6 — Onboarding + polish1.5 h
  • 3-card onboarding via TabView + PageTabViewStyle.
  • Empty states, loading skeletons (use .redacted(reason: .placeholder)).
  • Haptics on report submit + pin drop (UINotificationFeedbackGenerator).
  • Launch screen with the pomegranate mark.
Phase 7 — Stretch: push notificationsonly if time
  • Supabase Edge Function triggers on new active outage, sends APNs via Apple's HTTP/2 API.
  • Device token registered on first sign-in.
  • Notification tap deep-links to the outage detail.

Quality bar

  • Real Azerbaijani everywhere. No machine translation. If unsure, leave the key and ask.
  • Mobile-perfect. Test on iPhone SE (smallest) and iPhone 15 Pro Max. Map must work in both orientations.
  • Offline resilience. Cache the last successful outage fetch; show it with a "Bağlantı yoxdur — son yenilənmə X dəq əvvəl" banner if the network drops.
  • Privacy. Add NSLocationWhenInUseUsageDescription, NSCameraUsageDescription, NSPhotoLibraryUsageDescription — all in Azerbaijani, all explaining the real reason.
  • No third-party analytics. Not for v1. Privacy is part of the pitch.

Deliverables at the end

  • Working app installed on a physical iPhone, signed with a free Apple ID for demo.
  • Short README with: prerequisites, how to set the Supabase keys, how to run.
  • 30-second screen recording as backup demo footage.
  • App icon — pomegranate mark on the dark navy background, exported at all required sizes.

This brief is for the Openwave 48-hour hackathon (İRİA × Nərimanov RİH × Tedspace, 2026). The web app is the primary submission; this iOS app demonstrates platform reach and serves real residents on the device they actually carry. Build accordingly.
